# タスク記録フォーマット

## テンプレート

```markdown
# タスク記録: [タスク名]

**完了日時**: YYYY-MM-DD HH:MM

## 概要
このタスクで何を達成したかを簡潔に説明

## 実施内容
### 作業項目
- [ ] 実施した具体的な作業内容1
- [ ] 実施した具体的な作業内容2
- [ ] 実施した具体的な作業内容3

### 変更したファイル
- `ファイルパス1` - 変更内容の説明
- `ファイルパス2` - 変更内容の説明

### 追加・作成したファイル
- `ファイルパス1` - ファイルの目的と内容
- `ファイルパス2` - ファイルの目的と内容

## 技術的詳細
### 実装のポイント
- 重要な実装ポイント1
- 重要な実装ポイント2

### 使用した技術・ライブラリ
- 技術名1: 使用目的
- 技術名2: 使用目的

### 設定・パラメータ
- 設定項目1: 値 (理由)
- 設定項目2: 値 (理由)

## 結果・確認事項
### 動作確認
- [ ] 確認項目1
- [ ] 確認項目2
- [ ] 確認項目3

### テスト結果
- テスト項目1: 結果
- テスト項目2: 結果

## 課題・注意事項
### 残課題
- 課題1: 詳細と対応予定
- 課題2: 詳細と対応予定

### 注意点
- 今後の作業で注意すべき点1
- 今後の作業で注意すべき点2

## 参考資料
- [資料名1](URL) - 説明
- [資料名2](URL) - 説明

## 次のステップ
- 次に実施すべきタスク1
- 次に実施すべきタスク2
```

## 使用方法

1. 新しいタスクを開始する前に、このテンプレートをコピーして新しいマークダウンファイルを作成
2. ファイル名は `YYYY-MM-DD_task-name.md` の形式で命名
3. タスク完了後、各セクションを埋めて記録を完成させる
4. `task_log/` ディレクトリに保存

## 記録のガイドライン

### 記述のポイント
- **具体性**: 「設定を変更した」ではなく「ポート番号を8080から3000に変更した」
- **理由の明記**: なぜその変更を行ったかの理由も記載
- **再現性**: 他の人が読んでも同じ作業ができるレベルで詳細に記述
- **問題と解決**: 遭遇した問題とその解決方法も記録

### 必須記載事項
- 完了日時
- 変更したファイルとその内容
- 動作確認の結果
- 残課題（ある場合）

### オプション記載事項
- スクリーンショット（必要に応じて）
- ログファイルの抜粋
- エラーメッセージと解決方法
- パフォーマンス測定結果
