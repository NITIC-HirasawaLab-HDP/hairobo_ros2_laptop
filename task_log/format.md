# タスク記録フォーマット

## テンプレート

```markdown
# タスク記録: [タスク名]

**完了日時**: YYYY-MM-DD HH:MM

## 概要
このタスクで何を達成したかを具体的かつわかりやすく説明(表や絵文字を使っても構わない)
初学者でもわかりやすく説明(このセクションが最も大事)

## 課題・注意事項
### 残課題
- 課題1: 詳細と対応予定
- 課題2: 詳細と対応予定

### 注意点
- 今後の作業で注意すべき点1
- 今後の作業で注意すべき点2

## 参考資料
- [資料名1](URL) - 説明
- [資料名2](URL) - 説明

## 次のステップ
- 次に実施すべきタスク1
- 次に実施すべきタスク2
```

## 使用方法

1. 新しいタスクを開始する前に、このテンプレートをコピーして新しいマークダウンファイルを作成
2. ファイル名は `YYYY-MM-DD_task-name.md` の形式で命名
3. タスク完了後、各セクションを埋めて記録を完成させる
4. `task_log/YYYY-MM-DD` ディレクトリに保存

## 記録のガイドライン

### 記述のポイント
- **具体性**: 「設定を変更した」ではなく「ポート番号を8080から3000に変更した」
- **理由の明記**: なぜその変更を行ったかの理由も記載
- **再現性**: 他の人が読んでも同じ作業ができるレベルで詳細に記述
- **問題と解決**: 遭遇した問題とその解決方法も記録

### 必須記載事項
- 完了日時
- 変更したファイルとその内容
- 動作確認の結果
- 残課題（ある場合）

### オプション記載事項
- スクリーンショット（必要に応じて）
- ログファイルの抜粋
- エラーメッセージと解決方法
- パフォーマンス測定結果
